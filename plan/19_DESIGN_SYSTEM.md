# 1. Purpose

This document defines the visual design standards for
the application.

The objective is to maintain a consistent, modern, and
professional appearance across every screen.

---------------------------------------------------------

# 2. Design Philosophy

The interface should resemble a lightweight
cybersecurity dashboard.

It should feel:

• Clean

• Professional

• Modern

• Responsive

• Easy to understand

---------------------------------------------------------

# 3. Theme

Default Theme

Dark Mode

Future Support

Light Mode

---------------------------------------------------------

# 4. Color Palette

Primary

Blue

Success

Green

Warning

Orange

Error

Red

Background

Dark Gray

Cards

Gray

Text

White

Secondary Text

Light Gray

---------------------------------------------------------

# 5. Typography

Primary Font

Segoe UI

Fallback

Arial

Headings

Bold

Body Text

Regular

Use consistent font sizes throughout the application.

---------------------------------------------------------

# 6. Buttons

Primary Button

Blue

Secondary Button

Gray

Danger Button

Red

Disabled Button

Dark Gray

Buttons should have consistent padding and rounded
corners.

---------------------------------------------------------

# 7. Cards

Dashboard cards should display:

Icon

Title

Value

Optional description

Cards should have equal spacing and alignment.

---------------------------------------------------------

# 8. Tables

Every table should include:

Column headers

Alternating row colors

Scrollable content

Row highlighting on selection

Columns should resize appropriately.

---------------------------------------------------------

# 9. Charts

Use simple and readable charts.

Supported:

Line Chart

Bar Chart

Pie Chart

Charts should update only when underlying data changes.

---------------------------------------------------------

# 10. Icons

Use simple, recognizable icons.

Suggested categories:

Dashboard

Network

Scanner

Packets

Reports

Settings

About

Exit

Maintain a consistent icon style.

---------------------------------------------------------

# 11. Notifications

Notification Types

Success

Information

Warning

Error

Notifications should be short and disappear
automatically after a few seconds.

---------------------------------------------------------

# 12. Dialog Boxes

Used for:

Confirmation

Errors

Warnings

Information

Dialogs should clearly explain the action required.

---------------------------------------------------------

# 13. Spacing

Maintain consistent margins.

Maintain equal spacing between controls.

Avoid overcrowded layouts.

---------------------------------------------------------

# 14. Responsiveness

Support common desktop resolutions.

Adjust tables and charts automatically when the window
is resized.

---------------------------------------------------------

# 15. Accessibility

Readable font sizes.

High contrast colors.

Clear labels.

Visible button states.

Simple navigation.

---------------------------------------------------------

# 16. Consistency Rules

The same action should always look and behave the same
throughout the application.

Avoid mixing different visual styles.

---------------------------------------------------------

# 17. Final Design Principle

A user should immediately recognize the application as a
professional cybersecurity tool.

The design should prioritize clarity and usability over
visual complexity.
